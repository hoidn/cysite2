## 3. TODO System Upgrade

### Current System

Uses `.notice` class — orange left-border boxes that look like site design elements. Problems:
- No indication of which source file to edit
- Looks like part of the site rather than an editing instruction
- Inconsistent usage (some are genuine notices, some are TODOs)

### New System

Adopt the approach from website2: yellow background callout boxes with a monospace file-path badge.

**CSS classes:**
- `.todo` — Yellow background (#fff8e1), left border (4px solid #f9a825), padding, border-radius
- `.todo__file` — Inline monospace badge showing the source file path (e.g., `index.html`)
- `.todo__text` — The instruction text

**HTML pattern:**
```html
<div class="todo">
  <span class="todo__file">index.html</span>
  <p class="todo__text">
    Add a paragraph about your current Scripps research.
  </p>
</div>
```

### Where TODOs Should Appear

| File | TODO |
|------|------|
| `index.html` | Add paragraph about current Scripps research in About section |
| `index.html` | Add forward-looking research vision paragraph for faculty applications |
| `research.html` | Write Air-Sea Interaction theme describing current postdoc work |
| `research.html` | Write 2-4 sentence research vision statement connecting PhD and postdoc |
| `research.html` | Revise Future Directions to reflect actual research trajectory |
| `cv.html` | Replace placeholder with actual CV PDF link |
| `cv.html` | Add dissertation title, additional degrees, employment dates |
| `cv.html` | Add conference presentations (6 APS DFD talks, 2016-2024) |
| `cv.html` | Add awards (APS DFD 2021 poster award) |

### Retire `.notice` Class for TODOs

Keep `.notice` only for genuine site notices (e.g., "Publications list reflects available materials"). Convert all placeholder/editing instructions to `.todo`.
